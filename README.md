# IT-STARTS-TODAY-

This repository is connected to your **audos** workspace. Every push from
audos lands here, and (when two-way sync is on) pushes you make here can be
pulled back into audos.

> **Heads up:** `README.md` and `AUDOS.md` are generated by audos and are
> rewritten on every push. If you edit them in this repo your changes will be
> overwritten — change your workspace instead.

## Where your workspace lives in this repo

All of your workspace files are written inside the **`audos-workspace-675942/`** folder.
Anything outside that folder is ignored when audos pulls changes back in.

| In this repo (`audos-workspace-675942/…`) | In the audos UI | What it is |
| --- | --- | --- |
| `Desktop.tsx` | Your Space (the desktop) | The shell that draws your dock, app windows, app launcher, and Otto chat. |
| `config.json` | Space settings / app list | The manifest: which apps exist, branding, theme, layout. It is the source of truth for what shows up in your Space. |
| `apps/<AppName>/App.tsx` | Apps in your Space (dock + launcher) | A code-first app. Each one is registered by an entry in `config.json`. |
| `landing-pages/*.tsx` | Product Studio → Landing Pages (microsites) | Single-file landing pages, compiled into app records. |
| `mini-apps/*.tsx` | Product Studio → Mini Apps | Single-file mini-apps, compiled into app records. |
| `SpaceRuntimeContext.tsx` | (runtime) | Shared React context / runtime logic for your Space. |
| `components/` | (building blocks) | Shared React components you can import from `Desktop.tsx` or your apps. |
| `hooks/`, `lib/`, `data/`, `assets/` | (building blocks) | Shared hooks, utilities, JSON data, and static assets. |
| `tools/` | Otto's capabilities | Agent tools Otto can call. |

## How it compiles (and the one gotcha)

audos compiles your workspace on the backend with **esbuild** (JSX automatic
runtime, ESM output, with React / ReactDOM / Lucide resolved from a CDN
importmap — so you never bundle React yourself).

- **`Desktop.tsx`** is the entry point for your Space. It is compiled together
  with the files it imports.
- **Apps are config-driven.** An app only appears in your Space if it has an
  entry in `config.json` (its `component` field points at the source file, e.g.
  `apps/Tasks/App.tsx`). Dropping a `.tsx` file under `apps/` *without* a
  matching `config.json` entry will **not** make it show up.
- **`components/` are not wired in automatically.** Adding or editing a file in
  `components/` does **not** make it appear inside `Desktop.tsx` or any app on
  its own — something has to `import` it. Editing `components/Foo.tsx` only
  changes the screens that already import `Foo`.

## What you can edit vs. what audos manages

**Safe to edit** — these round-trip through sync:
`Desktop.tsx`, `config.json`, `SpaceRuntimeContext.tsx`, and your files under
`apps/`, `components/`, `hooks/`, `lib/`, `data/`, `assets/`,
`landing-pages/`, and `mini-apps/`.

**Platform-managed — don't edit here, audos regenerates them:**
- `README.md` and `AUDOS.md` (rewritten on every push).
- `*.generated.ts` / `*.generated.css` (e.g. `theme.generated.*`,
  `branding.generated.css`) — regenerated from your branding / theme settings.
- Anything under `.published-source/` (compiler output).

## How the two directions work

**audos → GitHub (Push to GitHub)**
audos writes the files exactly as they exist in your workspace. We only write
`config.json` fields you (or the agent) actually set — anything left at a
platform default does **not** appear in `config.json`, so the file can look
smaller than you expect. To override a default, just add the field with the
value you want.

**GitHub → audos (Sync from GitHub)**
Whatever is in this repo wins. If you delete a key from `config.json` or remove
a file, that deletion lands in audos on the next sync. This is the direction to
use when you want to *remove* something the agent added.

Commits made by audos are tagged `[audos-sync]` so you can spot them in your
git history, and audos skips its own `[audos-sync]` commits on the way back in
to avoid loops.

## Switching off the sync

Disconnect the repo from the audos developer panel, or change the dev mode to
`platform` (audos-only) or `external`.
